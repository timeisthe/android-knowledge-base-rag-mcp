# 结论先行

近两年的 Android 面试已经不是“背四大组件和 Handler 就够了”。稳定的考察结构是：

项目真实性与指标 → Java/Kotlin 与并发基础 → Android Framework 机制 → 性能与工程问题 → 场景设计 → 算法或代码实现。

不同公司权重不同，但“从简历中的一句技术描述连续追问到底”几乎是共同模式。

# 三层准备优先级

## P0：没有这些，面试很容易在前 20 分钟结束

1. Java 集合与并发
   - HashMap 的结构、put/resize、可变键风险、线程安全问题。
   - synchronized、volatile、Lock、CAS、ConcurrentHashMap。
   - 线程池参数、拒绝策略、任务排队和安全停止。

2. Android 核心机制
   - Activity/Fragment 生命周期、任务栈、状态恢复。
   - Handler、Looper、MessageQueue 和阻塞唤醒。
   - Binder/IPC 的选择理由和一次调用的基本路径。
   - View 绘制、事件分发、RecyclerView 复用。

3. 网络
   - TCP 可靠性、三次握手和四次挥手。
   - HTTPS/TLS 握手，HTTP 1.1/2/3 的核心差异。
   - OkHttp 拦截器、连接池、缓存和同步/异步请求。

4. 内存
   - GC Roots 和可达性分析。
   - Android 常见泄漏引用链。
   - LeakCanary 的检测点、对象观察和堆分析。

5. 算法
   - 链表、二叉树、哈希、双指针、DFS/BFS、LRU。

## P1：决定能否从“会答八股”进入“能做工程”

- 启动优化：基线、关键路径、初始化编排、首帧、异步边界。
- 列表性能：复用、绑定成本、图片/视频、Diff、预取、埋点。
- 数据与状态：Room、缓存、StateFlow、ViewModel、进程重建。
- 项目证据链：问题、指标、方案、对照实验、回滚和线上验证。
- 设计模式：必须能落到 Android/OkHttp/Glide/业务模块的实际关系。

## P2：区分大厂、核心业务或社招深度

- Framework 源码边界：AMS、Binder、nativePollOnce、主线程启动。
- 跨端渲染与通信：Flutter、RN、KMM、JNI、Unity。
- 大型场景诊断：首帧慢、数据乱序、旧状态、错跳、埋点与真实展示不一致。
- 动态化、插件化、热更新、包体和混淆。
- 线上性能与可观测性：trace、heap、火焰图、日志关联、实验分桶。

# 面试轮次的典型变化

## 一面

更看重基础覆盖面和简历真实性。常见组合是：

- 一个项目切入；
- Java/Android/网络基础连续追问；
- 一道链表或树题；
- 确认到岗时间和岗位动机。

## 二面

从“知道什么”转为“能否把系统设计清楚”。常见变化：

- 深挖项目数据结构和边界条件；
- 给业务场景，要求拆数据、状态、线程、生命周期和容错；
- 要求解释优化指标和线上验证；
- 算法更重边界、复杂度和可维护性。

## 三面或社招

更关注 owner 能力：

- 如何定位线上偶现问题；
- 如何组织跨模块改造和灰度；
- 方案成本、风险、回滚和观测；
- 技术广度是否能跨 JNI、跨端、硬件或服务端协作。

# 高频追问链

回答一个名词通常只是开始。典型链条如下：

- Handler → Looper 属于谁 → MessageQueue 如何阻塞 → 新消息如何唤醒 → 子线程如何创建 → 为什么会泄漏。
- HashMap → 冲突处理 → 扩容 → 树化条件 → 为什么不用 AVL → 并发下哪里坏 → ConcurrentHashMap 如何改进。
- ViewModel → 为什么比 Activity 活得久 → 配置变更如何保留 → 进程死亡能否保留 → SavedStateHandle 的边界。
- HTTPS → 对称/非对称分别做什么 → 证书链 → 如何校验 → 连接复用 → HTTP/2、HTTP/3。
- 启动优化 → 指标是什么 → 哪段关键路径最慢 → 初始化如何编排 → 异步后如何保证依赖 → 如何证明没有功能回归。
- 内存泄漏 → 引用链 → GC Roots → LeakCanary 何时观察 → 堆快照怎么看 → 修复后如何验证。

# 最实用的准备原则

- 每个知识点准备“定义、为什么存在、核心数据结构、执行流程、边界、工程例子”六部分。
- 简历上的每个技术词都要能承受至少三层追问。
- 性能优化必须有前后指标和采集方法，不能只说“异步、缓存、懒加载”。
- 场景题先明确需求与约束，再拆数据流、状态、线程、生命周期、失败恢复和观测。
- 算法写完必须主动给复杂度、空输入、重复值、溢出和并发边界。
