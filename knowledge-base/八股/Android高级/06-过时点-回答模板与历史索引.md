# Android 高级八股：过时点、回答模板与历史索引

> 时效性：2026-07-23 使用 Android CLI 1.0.15857036 与 Android 官方知识库复核。版本敏感内容已在正文标注；第三方旧链接仅作历史索引。

## 27. 常见过时点与当前口径速查

| 旧口径 | 当前建议 |
| --- | --- |
| Fragment 只背一条生命周期回调链 | 区分 Fragment Lifecycle 与 View Lifecycle；ViewPager2 离屏页通常最高 `STARTED` |
| 在 Fragment 本体生命周期观察 View 数据 | 使用 `viewLifecycleOwner`，在 `onDestroyView()` 释放 ViewBinding |
| `onSaveInstanceState` 顺序固定 | API 28 前后与 `onStop` 顺序不同；不要依赖固定序列 |
| ContentProvider ANR 固定约 10 秒 | 阈值由远程调用检测设置决定，需看具体调用链 |
| 广播永远 10/60 秒 | Android 14+ CPU 饥饿时可扩到 10–20/60–120 秒 |
| IntentService/AsyncTask | 新代码优先协程、WorkManager；持续用户可感知任务才考虑前台服务 |
| 前台服务可长期后台运行 | Android 14 要求类型和权限；Android 15 部分类型累计 6 小时/24 小时并回调超时 |
| 动态 Receiver 不关心导出属性 | API 33 有导出 flag；面向 Android 14+ 的非纯系统广播必须明确 |
| LocalBroadcastManager/粘性广播 | 已废弃或不推荐，应用内状态用明确回调或 Flow |
| ViewPager + FragmentPagerAdapter | 新代码优先 ViewPager2 + FragmentStateAdapter |
| 直接覆写 `onBackPressed()` | 使用 OnBackPressedDispatcher/Navigation，兼容预测性返回 |
| 自定义 scheme 等同可信链接 | 网站到 App 优先已验证 App Links；Android 15+ 可用 Dynamic App Links |
| `rememberSaveable` 可保存任意状态 | 仅保存 Bundle 兼容的小型 UI 状态；业务数据从数据层恢复 |
| Transform API | AGP 8.0 已移除，使用 `androidComponents` Instrumentation API |
| Baseline Profile 与 Startup Profile 是一回事 | 前者优化关键代码预编译，后者优化启动 DEX 布局，两者互补 |
| ART 只 AOT | 现代 ART 结合解释、JIT、AOT 与 Profile |
| 传统插件化/热修复是常规默认方案 | 更多作为原理考点；工程上优先组件化、动态特性、配置化并遵守商店政策 |

## 28. 面试回答模板

遇到 Framework 原理题，可以按这个模板回答：

1. 先说明这个机制解决的问题。
2. 说出核心对象和调用链。
3. 强调线程/进程切换点。
4. 补充版本差异或限制。
5. 结合项目实践讲排查或优化。

示例：问 Handler 原理。

可以回答：Handler 是 Android 的线程消息机制，用于把任务投递到某个 Looper 所在线程执行。一个线程通过 ThreadLocal 绑定一个 Looper，Looper 持有 MessageQueue；Handler 发送 Message 到队列，Looper 循环取出消息后根据 Message.target 分发回对应 Handler。主线程 Looper 在 ActivityThread.main 中创建，没有消息时 MessageQueue 会阻塞等待，所以 loop 死循环不会空转。实际使用要注意延迟消息导致的 Activity 泄漏，现代项目里很多场景可以用 lifecycleScope 或 viewModelScope 替代手写 Handler。
