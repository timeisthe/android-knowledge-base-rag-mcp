# Android 高级八股：2026 时效性审校说明

## 范围

本分类由《Android高级开发面试八股优化整理.md》内化而来，覆盖 Activity、四大组件、Handler、View、RecyclerView、Fragment、WebView、Bitmap、架构、Binder、性能、系统启动、APK、ART、组件化、Jetpack、网络与协程。

本次审校使用 Android CLI 1.0.15857036 检索并抓取 Android 官方知识库，审校日期为 2026-07-23。稳定的 Framework 原理保留；与 targetSdk、AndroidX、AGP 和系统版本相关的结论按当前文档改写。

## 关键修订

| 主题 | 修订后的当前口径 |
| --- | --- |
| Fragment | 同时存在 Fragment 与 View 两套 Lifecycle；上行与下行的事件/回调顺序不同；离屏 ViewPager2 Fragment 通常最高 STARTED |
| Activity SavedState | 不依赖固定回调序列；API 28 前后 `onStop` 与保存状态的先后不同 |
| ANR | Android 14+ 广播超时可因 CPU 饥饿扩展；ContentProvider 不再统一背 10 秒 |
| 前台服务 | Android 14 要求类型/权限；Android 15 的 dataSync/mediaProcessing 在后台受 6 小时/24 小时累计限制 |
| BroadcastReceiver | API 33 引入导出 flag；面向 Android 14+ 的动态非系统 Receiver 必须明确导出行为 |
| Deep Link | 网站链接优先已验证 App Links；Android 15+ 支持服务端 Dynamic App Links |
| WebView | JS bridge 必须限制可信内容与 origin；WebMessage 也必须校验 target origin |
| 性能 | Baseline Profile 与 Startup Profile 分工不同，应使用 Macrobenchmark/Perfetto 验证 |
| AGP 字节码插桩 | Transform API 自 AGP 8.0 移除，使用 androidComponents Instrumentation API |
| Compose | rememberSaveable 只放 Bundle 兼容的小状态；Flow 使用 collectAsStateWithLifecycle |
